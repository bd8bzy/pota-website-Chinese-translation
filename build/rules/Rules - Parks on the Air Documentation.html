    规则

    版本 日期 描述
    1.0 2022年8月 从原网站迁移过来并做了显著重构

    ❗重要： 因参与空中公园（Parks on the Air®，POTA）所导致的受伤及死亡风险由你自行承担。 你必须遵守你无线电执照的限制及国家和当地的现行法律法规；这些规定高于这些文档中的一切内容。

    基本上POTA是这么一项活动：业余无线电爱好者来到公园，在里面发起与其它任何地方的业余无线电爱好者的通联。下面的详细描述是对这个定义的进一步阐释。

    关键定义

    • 激活人：激活人（activator）是指一名持照业余无线电操作员，身处某个POTA列表所指定的公园内，发起与其它持照业余无线电操作员的通联。
    • 猎人：猎人（hunter）是指其它持照业余无线电操作员，与身处公园的激活人进行通联。有时也会使用“追逐者”的术语来指代。
    • 激活：激活（activity）是指激活人进入一个公园，发起POTA活动以激活该点位。

    激活 / 尝试

    1. 要成功激活，必须在一个 UTC 日（Zulu 日）内从指定列表中的公园完成至少 10 个 QSO。
      • 有礼貌的激活人仍会提交未成功激活的日志，以确保 Hunter 能得到 QSO 的分数。
      • 同一个UTC天内，在同一个省市区的同一个公园内的多条激活，计数为单一一条激活，前提是有了10条以上 QSO。

    合格的 QSO

    1. 波段/模式： 只要符合本文档日志章节的要求，参与者所照准的所有波段和模式都可以用于POTA。
      • 注意： POTA并不是一项竞赛；POTA 的 QSO 可以出现在包括WARC(30m/17m/12m)在内的任何业余无线电波段上。
    2. 不允许地面中继：通过地面中继进行的 QSO 在 POTA 里不被承认。
      • 请参看术语表以获取 POTA 对于地面中继的定义。
      • 你可以使用地面中继进行直接通联，但激活人作为分数项提交上来的 QSO 不能经由地面中继。
    3. 允许卫星中继：通过卫星进行通联可被承认。
      • 请参看术语表以获取 POTA 对于卫星中继的定义。
      • 提交的日志里应当包括激活人发射频率的波段信息。
    4. 点位：你可以自己在 POTA 的页面上标注点位；任何人（包括猎人） 也可以来标注/重新标注你，无论他们是否在跟你通联。
      • 页面上的点位信息约在最后一次打点或标注为QRT的30分钟后消失。
      • 如果你在激活公园的同时，也在参与另一项通联活动，例如 ARRL的野架日活动，那么这些活动的规则有可能是不允许自我标注点位的。请遵循这些活动的要求。
    5. 功率限制： POTA 并不设置功率限制，但请你依然遵守照准等级/波段关于发射频率的法规，并且在能满足通联时请保持最低发射功率。
    6. 禁止全自动通联：一次通联必须包含双方操作员的直接参与操作。全自动通联是不被允许的。

    位置的激活及访问

    1. 必须在 POTA 指定列表 中的公园进行激活，而且这些公园当前必须对公众开放。
      • 当一个公园对居民合法开放，或者一个公共区域得到特许被划归到一个地图中并且可达，那么该公园就被视作是开放的。访问该公园时请遵循任何相关机构/相关网站所提及的限制要求。
      • 当相关机构/相关网站明确指示某公园已不再面向公众开放，或者其不再存在时，那么该公园就被视作是关闭的。
      • 一些公园里的设施、特许地、场所、大门等可能会出现季节性关闭，但公园其它部分可能还是开放的。如有不确定请联系公园相关办事机构办事人员。
    2. 激活人及其所有设备都必须位于公园界限内的公共场所。
      • POTA 所提供的公园地图仅供参考。请参看园区相关官方网站/官方机构以获取公园的精确官方定界。
      • 由停在公园界限内的各种车辆、房车上发起的激活是被允许的。
      • 由位于公园界限上空的飞行器所发起的激活是被允许的。
    3. 激活人在访问公园途中请勿擅入擅穿私有领地。
    4. 激活人不得在私有领地上发起激活，哪怕该领地毗邻或位于公园内。
    5. 如果某个路径、河流等已作为某个公园内的 POTA 实体被单独划定（而不是该公园本身），那么在激活时，操作员及其设备距离该实体边缘必须小于30.5米（100英尺）。
    6. POTA 中允许多重（“多点位”、“一送一”、“一送二”等）激活。
      • 每个激活都必须遵循上述点位激活及访问的要求，而且多点位激活时要求操作员及其设备必须整体性全部位于这几个 POTA 实体的界限所重合的区域内。
      • 所同时激活的每个公园都必须单独提交上传日志。

    猎人位置

    1. 猎人可以从家里或其它任何地方发起对激活人的通联。
      • 如果猎人恰巧也位于 POTA 指定列表的公园内，那么此次 QSO 就变成了“公园对公园”（P2P）通联。除开需要遵循的通用规则，请参看我们的 公园对公园 文档页以获取详情。

    日志要求

    1. 只有按照下述要求提交的通联日志，才能视作是有效的激活。
      • 猎人无须为 POTA 提交日志，只有激活人才需要提交。猎人通过激活人提交的激活日志获取分数。
    2. 日志的提交并不设截止时间。
      • 有礼貌的操作员不会拖延太久，因为猎人的分数/奖项都取决于他的日志提交。
    3. 延展多天的单个日志应当仅包含一个公园点位（无论是某公园有多个点位，还是同点位多公园多重激活）。本要求也适用于俱乐部及集体台激活。
    4. 除开本节末尾单独指明的情况外，日志文件必须是 ADIF 格式。
      • 请查看 激活人参考 以了解我们是如何使用 ADIF 中的各种字段。
      • 必填字段：
        • <STATION_CALLSIGN> 或 <OPERATOR>
          • 俱乐部/集体台日志的 QSO 条目中必须同时含有 <STATION_CALLSIGN> 及 <OPERATOR> 字段。<STATION_CALLSIGN> 字段包含了俱乐部/集体台的呼号。
          • 个人日志 QSO 中这两个字段的值必须相同，或者填写二者之一。
        • <CALL>
        • <QSO_DATE>
          • QSO 的 UTC 日期
        • <TIME_ON>
          • QSO 的 UTC 时间
        • <BAND>
        • <MODE> 或 <SUBMODE>
          • POTA 只存一个模式字段。如果 MODE 及 SUBMODE 字段都提交了，会使用 SUBMODE 字段。
      • 可选字段：
        • <MY_SIG>
        • <MY_SIG_INFO>
        • <SIG>
        • <SIG_INFO>
        • <MY_STATE>
      • POTA 所能够接受的 ADIF 文件特例
        • 在亚毫米波通联中，POTA 在<BAND> 字段中接受 SUBMM
    5. 俱乐部或集体台日志必须以该俱乐部/集体名义注册的 POTA 账户来提交，而不能通过个人账户提交。
    6. 日志里的个人 QSO 会被拒绝。
      • 上传工具会做日志去重。
      • POTA 每条 QSO 的唯一性由下述信息一起确定：
        • 上传该日志的账户 (也就是该<STATION_CALLSIGN>呼号的拥有者)下, <CALL>, <MODE>, <QSO_DATE>, <BAND>, <MY_SIG_INFO>, <SIG_INFO>, 位置 (如果公园跨省市区，则在上传时进行选择)
      • <CALL> 可能跟 <STATION_CALLSIGN> 或 <OPERATOR>不一样(也就是台或操作员无法自己完成)
      • 不能提交未来的 QSO

    💡 注意：在本规则中的某些信息跟本站其它地方（指南等）的信息冲突时，请以本规则为准。如发生了这种冲突，请发邮件 help@parksontheair.com 知会我们。

    附加说明 / 指导

    空中公园并不须要额外的 QSO 交互。

    可添加到项目中的公园、路径、河流的资质要求

    1. 新公园必须是国家/省/市/区县所有。POTA 不包括那些私有机构/地方政府所运营、所部分持有的、所资助的公园。
    2. 想要被包含进 POTA 的公园，必须满足上文所有要求，并且须要有个网站能提供其地图边界信息。
    3. POTA 不允许在已注册的公园内创建子公园条目（这些公园归属于国家/省/市/区县的同一个管理机构）。

    报告违反规则的情况

    POTA 是一项自我约束、社区共同监督的活动。如果你观察到某些违反规则的情况，可以报告给 help@parksontheair.com 以进行跟进。在采取任何行动之前，你必须提供足够的详细信息以毫无疑问地证明发生了违规行为。

    公园的访问及信息

    如果列出的某个公园已被永久关闭、根据本规则所描述的公众访问已无法实现、公园不再存在、你被告知不再允许在公园里进行通联、或者关于公园的任何新需要更新，你都可以报告到 help@parksontheair.com。

    方针、解释及目的

    POTA 将规则指定的这么简单，其目的就是为了让我们走出去、玩儿起来。对某些人来说，这些条目看起来仍然可能模棱两可。这些条目主要阐述了 POTA 管理员、开发者创立系统时的一些指导规范。未能符合这些指导也不见的会给你带来麻烦，但你（及通联的相关人）的统计数据和奖项可能就会跟你预期的不一样。

    公园界限和多公园激活

    请留意“激活人及其全部设备必须位于公园边界内的公众区域”这项规则的简单之美。如果一个激活人跨坐在公园界限上，那他就违反了这项规则。对于多公园激活也是一样，激活人及其全部设备必须位于每个公园界限的公共区域内，因此多重激活者所处的位置必须是这几个公园的公共交叠区域。

    俱乐部及集体台

    一般情况下，在有多个操作员时，个人不得在通联时使用彼此的呼号，而是应该使用协会或特殊活动的呼号。通联时应该使用协会或者特殊活动的呼号（就像野架日活动一样），并记录在 <STATION_CALLSIGN> 字段中。建立通联的个人应当在 <OPERATOR> 字段中记录他们的呼号（通常不在通联时使用）。只要上传日志到 <STATION_CALLSIGN> 持有人的账户上（参见 日志要求章节3）， 系统就会同时将奖项计入 <STATION_CALLSIGN> 和 <OPERATOR> 呼号的持有者的 QSO 上。

    请参阅 POTA 的 俱乐部指南 以了解我们对俱乐部/集体台激活所涉及的计划、行为规范及点位所作出的一些建议。

    传递麦克风

    非俱乐部情况下，若有多个人同时激活，操作员们可以“传递麦克风”轮流发起通联。这种情形里，每个操作员都需要按照日志要求 单独维护并提交自己的日志。

    为了俱乐部/集体台中残障人士顺利完成通联、为了传递麦克风顺利进行，在国家法规的框架内，可以对本规则做出最低限度的背离。
